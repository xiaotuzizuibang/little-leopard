## 常见乱码模式（如果出现，说明有问题）

### ❌ 问题1：占位符残留
```
The XTERM5X model uses XTERM8X mechanism
```
**原因**：术语恢复失败

### ❌ 问题2：HTML实体
```
这是&quot;引用&quot;的例子
```
**原因**：HTML实体未解码

### ❌ 问题3：字符重复
```
深深度度学学习习
```
**原因**：多次替换导致重复

### ❌ 问题4：术语被翻译
```
变压器模型使用多头注意
```
**原因**："transformer" 被错误翻译为"变压器"

### ❌ 问题5：空格异常
```
深度学     习    和   机器   学习
```
**原因**：零宽字符或空格未清理

## 调试方法

### 1. 查看控制台日志
1. 右键点击扩展图标
2. 选择"检查弹出内容"
3. 打开 Console 标签
4. 查找 `🐆` 开头的日志

**关键日志：**
```
🐆 Translation request: { text: "...", sourceLang: "auto", targetLang: "zh-CN" }
🐆 Protected English terms
🐆 Sending parallel requests to all translators...
🐆 Translation successful in XXXms using Google!
🐆 Original: The transformer model...
🐆 Translated: Transformer模型...
```

### 2. 检查缓存状态
如果翻译结果显示 `⚡` 符号，说明使用了缓存。清除缓存方法：
- 关闭并重新打开扩展
- 等待1小时缓存自动过期
- 重新加载扩展

### 3. 测试不同翻译提供商
查看日志中的 `provider` 字段：
- `Google` - Google Translate
- `MyMemory` - MyMemory API
- `LibreTranslate` - LibreTranslate API

不同提供商可能有不同的表现。

## 性能检查

### 正常指标
- ⏱️ 翻译时间：500ms - 3000ms
- 📊 缓存命中：第二次翻译相同内容应显示 `⚡`
- ✅ 成功率：>95%

### 异常指标
- ⚠️ 超时：>8000ms
- ❌ 失败：显示 "All translation services are currently unavailable"
- 🐌 过慢：>5000ms
