# 🔧 翻译乱码问题 - 修复说明

## 问题原因分析

翻译结果出现乱码可能由以下几个原因导致：

### 1. **术语保护占位符问题**
- **问题**：术语保护时使用的占位符 `__TERM_N__` 可能被翻译API识别为需要翻译的内容
- **影响**：占位符被部分翻译，恢复时无法正确匹配
- **修复**：改用更独特的占位符格式 `XTERMX`，不易被翻译

### 2. **HTML实体编码**
- **问题**：某些API返回HTML实体（如 `&quot;`、`&amp;`、`&#39;` 等）
- **影响**：显示为 `&quot;` 而不是 `"`
- **修复**：添加 `decodeHTMLEntities()` 函数自动解码

### 3. **零宽字符和特殊空白**
- **问题**：翻译结果可能包含零宽字符（U+200B-U+200D）
- **影响**：看不见但会影响文本显示和复制
- **修复**：使用正则表达式清除这些字符

### 4. **字符编码不匹配**
- **问题**：API请求/响应的字符编码不一致
- **影响**：中文显示为乱码
- **修复**：在所有API请求中添加 `Accept-Charset: utf-8` 头

### 5. **多次替换导致的冲突**
- **问题**：同一术语出现多次，占位符编号混乱
- **影响**：恢复时部分内容丢失或错位
- **修复**：改进占位符生成逻辑，使用回调函数确保唯一性

## 已实施的修复方案

### ✅ 修复1：改进术语保护逻辑 (`terminology.js`)

#### **变更前：**
```javascript
const placeholder = `__TERM_${this.counter}__`;
this.placeholders.set(placeholder, TERMINOLOGY[term.toLowerCase()]);
protectedText = protectedText.replace(match, placeholder);
this.counter++;
```

**问题**：
- `__TERM_5__` 这样的格式可能被部分翻译
- 多个匹配可能使用同一个占位符

#### **变更后：**
```javascript
protectedText = protectedText.replace(regex, (match) => {
  const placeholder = `XTERM${this.counter}X`;
  const translation = TERMINOLOGY[term.toLowerCase()];
  this.placeholders.set(placeholder, translation);
  this.counter++;
  return placeholder;
});
```

**改进**：
- ✅ 使用更简洁、不易被翻译的格式 `XTERM0X`, `XTERM1X`
- ✅ 回调函数确保每个匹配都有唯一占位符
- ✅ 立即返回替换值，避免二次替换

---

### ✅ 修复2：改进恢复逻辑 (`terminology.js`)

#### **变更前：**
```javascript
restoreTerms(text) {
  let restoredText = text;
  this.placeholders.forEach((term, placeholder) => {
    restoredText = restoredText.replace(new RegExp(placeholder, 'g'), term);
  });
  return restoredText;
}
```

**问题**：
- 正则表达式可能有特殊字符问题
- 没有排序，可能相互干扰

#### **变更后：**
```javascript
restoreTerms(text) {
  if (!text) return text;
  
  let restoredText = text;
  
  // 按占位符编号倒序恢复，避免占位符相互干扰
  const sortedPlaceholders = Array.from(this.placeholders.entries())
    .sort((a, b) => {
      const numA = parseInt(a[0].match(/\d+/)[0]);
      const numB = parseInt(b[0].match(/\d+/)[0]);
      return numB - numA;
    });
  
  sortedPlaceholders.forEach(([placeholder, term]) => {
    // 使用字符串分割再连接，更可靠
    restoredText = restoredText.split(placeholder).join(term);
  });
  
  return restoredText;
}
```

**改进**：
- ✅ 空值检查，避免错误
- ✅ 倒序恢复，避免编号相邻的占位符相互干扰
- ✅ 使用 `split().join()` 代替正则，更可靠

### ✅ 修复3：添加HTML实体解码 (`background.js`)

#### **新增函数：**
```javascript
function decodeHTMLEntities(text) {
  const entities = {
    '&amp;': '&',
    '&lt;': '<',
    '&gt;': '>',
    '&quot;': '"',
    '&#39;': "'",
    '&nbsp;': ' ',
    '&#x27;': "'",
    '&#x2F;': '/',
    '&apos;': "'"
  };
  
  return text.replace(/&[#\w]+;/g, (entity) => {
    return entities[entity] || entity;
  });
}
```

**作用**：
- 自动将 `&quot;` 转换为 `"`
- 处理所有常见的HTML实体编码

---

### ✅ 修复4：添加文本清理函数 (`background.js`)

#### **新增函数：**
```javascript
function cleanTranslationText(text) {
  if (!text) return text;
  
  // 解码HTML实体
  let cleaned = decodeHTMLEntities(text);
  
  // 移除多余的空白字符
  cleaned = cleaned.replace(/\s+/g, ' ').trim();
  
  // 移除零宽字符
  cleaned = cleaned.replace(/[\u200B-\u200D\uFEFF]/g, '');
  
  return cleaned;
}
```

**作用**：
- 综合处理所有文本格式问题
- 移除不可见的干扰字符
- 标准化空白字符

---

#### **所有API调用添加：**
```javascript
headers: {
  'Accept-Charset': 'utf-8',
  // ... 其他headers
}
```

**作用**：
- 明确指定UTF-8编码
- 确保中文正确传输

### ✅ 修复6：三重清理流程 (`background.js`)

#### **新的处理流程：**
```javascript
// 1. 清理API返回的原始结果
result.translation = cleanTranslationText(result.translation);

// 2. 恢复专业术语
result.translation = protector.restoreTerms(result.translation);

// 3. 再次清理（处理恢复后可能产生的格式问题）
result.translation = cleanTranslationText(result.translation);
```

**作用**：
- 三道防线确保文本质量
- 在术语恢复前后都进行清理

## 已知限制

### 1. **极长文本**
- **限制**：单次翻译最多1000字符
- **原因**：API限制
- **解决**：分段翻译

### 2. **特殊格式文本**
- **限制**：代码、公式可能格式错乱
- **原因**：翻译API不识别这些格式
- **解决**：手动保护这些内容

### 3. **生僻术语**
- **限制**：词典中没有的术语可能被翻译
- **原因**：词典有限（目前150+术语）
- **解决**：可以扩展词典
