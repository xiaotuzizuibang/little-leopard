# 🧠 术语系统说明

## 📁 文件结构

```
scripts/
└── terminology.js            # 统一术语系统（包含所有领域术语）
```

## 🎯 设计理念

### 统一管理
- **集中存储**: 所有术语集中在一个文件中，便于Chrome扩展加载
- **按领域分组**: 术语按领域逻辑分组，便于理解和维护
- **高效查找**: 优化的数据结构，快速匹配术语

### 智能保护
- **上下文感知**: 根据翻译模式选择相关术语
- **优先级匹配**: 长术语优先匹配，避免误替换
- **双向保护**: 支持中英文互译的术语保护

## 🔧 术语系统功能

### TerminologyProtector 类
```javascript
const protector = new TerminologyProtector();

// 保护英文术语（翻译成中文时）
const protectedText = protector.protectEnglishTerms(text);

// 保护中文术语（翻译成英文时）
const protectedText = protector.protectChineseTerms(text);

// 恢复术语
const restoredText = protector.restoreTerms(translatedText);

// 重置保护器
protector.reset();
```

### 术语数据结构
```javascript
// 英文到中文的术语映射
const TERMINOLOGY = {
  'deep learning': '深度学习',

// 中文到英文的反向映射
const REVERSE_TERMINOLOGY = {
  '深度学习': 'deep learning',

  // ... 更多术语

```

## 🚀 使用示例

### 在翻译中使用
```javascript
// background.js 中
const protector = new TerminologyProtector();

// 根据模式保护术语
const protectedText = protector.protectEnglishTerms(text, mode);

// 翻译后恢复术语
const finalText = protector.restoreTerms(translatedText);
```

### 添加新术语
```javascript
// 添加到特定领域
manager.addTerm('federated learning', '联邦学习', 'deep-learning-basics');

// 删除术语
manager.removeTerm('old term');
```

### 验证术语
```javascript
// 检查术语完整性
const issues = manager.validateTerminologies();
console.log(issues);
```

## 🔄 维护指南

### 添加新术语
1. 确定术语所属领域
2. 在对应的 `.js` 文件中添加
3. 更新 `scripts/terminology.js` 中的合并版本
4. 测试术语保护效果

### 修改现有术语
1. 在对应领域文件中修改
2. 同步更新合并版本
3. 验证翻译效果

### 新增领域
1. 创建新的领域文件
2. 在 `terminology-manager.js` 中注册
3. 更新翻译模式映射
4. 添加到合并版本

## 💡 最佳实践

1. **术语一致性**: 确保同一概念在不同领域中翻译一致
2. **长度优先**: 长术语优先匹配，避免短术语误替换
3. **上下文相关**: 根据翻译模式选择相关术语
4. **定期更新**: 跟随技术发展更新术语词典
5. **质量验证**: 定期验证术语翻译质量
