# 🔗 链接跳转功能测试指南

## ✅ 修复说明

**问题：** 点击标记的 🔗 按钮无法跳转到原网页

**原因：**
1. 使用了内联 `onclick` 属性，在扩展的 CSP（Content Security Policy）下被阻止
2. URL 中可能包含特殊字符（如 `'` `"`）导致 HTML 属性解析错误
3. 没有正确的 URL 转义处理

**解决方案：**
1. ✅ 改用事件委托（Event Delegation）而非内联事件
2. ✅ 将 URL 存储在 `data-marker-url` 属性中并转义
3. ✅ 使用 `chrome.tabs.create` API 打开链接（更可靠）
4. ✅ 提供多层降级方案：Chrome API → window.open → location.href

## 🐛 已知问题和解决方案

### **问题 1：点击后没有反应**

**可能原因：**
- 事件监听未正确绑定
- CSP 阻止了操作

**解决方法：**
1. 打开控制台查看错误
2. 刷新扩展
3. 重新打开 popup/history 页面

### **问题 2：跳转到错误的页面**

**可能原因：**
- URL 保存时有误
- URL 被错误转义

**解决方法：**
```javascript
// 查看实际保存的 URL
chrome.storage.sync.get(['markers'], (result) => {
  console.log('URLs:', result.markers.map(m => m.url));
});
```

### **问题 3：新标签页没有打开**

**可能原因：**
- 浏览器阻止了弹出窗口
- chrome.tabs API 权限问题

**解决方法：**
1. 检查浏览器的弹出窗口设置
2. 确认 manifest.json 中有 `tabs` 权限
3. 使用降级方案（右键打开链接）
